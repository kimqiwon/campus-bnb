## 1. 引言

### 1.1 编写目的
本文档描述"日常民宿预订管理系统"的功能与非功能需求，为后续设计、开发、测试提供依据。

### 1.2 项目背景
随着周边游和短租市场的发展，小型民宿经营者需要一个轻量级的管理系统来管控房源状态、处理租客预订、核算费用。本系统为单体民宿/小型民宿集群提供线上预订管理能力。

### 1.3 术语定义

| 术语 | 定义 |
|-----|------|
| 租客（Tenant） | 预订民宿房间的用户 |
| 房源（Room） | 可供出租的民宿房间 |
| 预订（Booking） | 租客对房源的预订记录 |
| 结算（Settlement） | 订单费用核算与核销归档 |
| 房型状态 | 空闲/已预订/维护中 |

## 2. 总体描述

### 2.1 产品视角
B/S架构，前后端分离。前端 Vue 3 + Element Plus，后端 Node.js + Express，数据库 MySQL 8.0。

### 2.2 功能模块概览

| 模块 | 功能 |
|-----|------|
| **租客模块** | 注册/登录、个人信息管理、查看预订记录 |
| **房源模块** | 房间信息录入、房型状态管控、图片上传 |
| **预订模块** | 提交订单、核对入住时间、查看空闲房源 |
| **订单结算模块** | 费用核算、订单核销、归档记录 |

### 2.3 用户角色

| 角色 | 描述 |
|-----|------|
| 租客 | 浏览房源、提交预订、查看订单 |
| 管理员（民宿老板） | 管理房源、确认预订、订单结算 |

## 3. 功能需求

### 3.1 租客模块（Tenant Module）

**FR-T01 - 租客注册**
- 输入：用户名、密码、真实姓名、手机号、邮箱
- 校验用户名唯一性、手机号格式
- 密码加密存储（bcrypt）

**FR-T02 - 租客登录**
- 输入：用户名 + 密码
- 返回 JWT Token，有效期 7 天

**FR-T03 - 查看个人信息**
- 展示：用户名、姓名、手机号、邮箱、注册时间

**FR-T04 - 查看预订记录**
- 展示该租客的所有历史预订
- 支持按状态筛选（待确认/已确认/已取消/已完成）

### 3.2 房源模块（Room Module）

**FR-R01 - 房间录入**
- 输入：房间标题、描述、价格/晚、房型（单人间/双人间/套房）、图片URL
- 默认状态：空闲

**FR-R02 - 房间列表查询**
- 支持按房型、价格区间筛选
- 只展示状态为空闲的房间（面向租客）

**FR-R03 - 房间状态管控**
- 管理员可修改房间状态：空闲 → 已预订（预订确认时自动）/ 维护中（手动）

**FR-R04 - 房间信息编辑/删除**

### 3.3 预订模块（Booking Module）

**FR-B01 - 提交预订**
- 选择房间、填写入住日期、退房日期
- 校验：退房日期 > 入住日期，且该时段房间未被预订
- 初始状态：待确认

**FR-B02 - 预订确认/取消**
- 管理员可确认预订（状态→已确认），自动将房间状态改为已预订
- 租客/管理员可取消预订（状态→已取消），房间状态恢复为空闲

**FR-B03 - 预订记录查询**
- 租客：查看自己的预订
- 管理员：查看所有预订

### 3.4 订单结算模块（Settlement Module）

**FR-S01 - 费用核算**
- 根据入住天数 × 每日价格自动计算总费用
- 预订确认后自动生成结算记录

**FR-S02 - 订单核销**
- 管理员标记订单为"已完成"
- 记录核销时间

**FR-S03 - 结算统计**
- 统计总收入、已完成订单数、取消率

## 4. 用例模型

### 4.1 租客用例

```
租客
├── 注册账号
├── 登录系统
├── 浏览房源列表
├── 提交预订申请
├── 查看我的预订
└── 取消预订
```

### 4.2 管理员用例

```
管理员
├── 登录系统
├── 录入房间信息
├── 编辑/删除房间
├── 管理房间状态
├── 查看所有预订
├── 确认预订
├── 取消预订
├── 费用核算
├── 订单核销
└── 查看结算统计
```
